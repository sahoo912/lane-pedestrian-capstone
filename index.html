Lane Detection & Pedestrian Posture Analysis

Software-based Capstone Project for Autonomous Driving

Problem Statement

This project develops a software system that detects lane boundaries in real-time using edge detection techniques and predicts pedestrian crossing intent based on posture analysis. The system works entirely in a simulated or video environment.

Software Architecture

  • Input: Pre-recorded datasets (KITTI, Cityscapes) or simulation (CARLA, Udacity).
  • Lane Detection: Canny edge detection → ROI masking → Hough Transform or polynomial fitting.
  • Pedestrian Detection: YOLOv8/SSD model for bounding boxes.
  • Posture Analysis: MediaPipe/OpenPose keypoint extraction → classification into standing, walking, crossing.
  • Decision Making: Rule-based system using lane + pedestrian info.
  • Output: Annotated video frames + simulated control commands.

Modules

  • Lane Detection Module: Detects road lanes using OpenCV edge detection & line fitting.
  • Pedestrian Detection Module: Detects people in the scene using deep learning models.
  • Posture Analysis Module: Classifies posture to predict movement intent.
  • Decision Module: Combines inputs to decide whether to maintain speed, slow, or stop.

Tech Stack

  • Python
  • OpenCV, NumPy
  • YOLOv8 / SSD
  • MediaPipe Pose / OpenPose
  • CARLA / Udacity Self-Driving Car Simulator
